<Callout type="intent" title="🎯 How to read this">
Complexities are **average case** unless noted. **Big-O** drops constants and lower-order terms — it describes how cost *grows*, which is what decides whether your code survives 10× the data.
</Callout>

## Complexity at a glance

| Class | Name | Example in Go |
|---|---|---|
| O(1) | constant | `m[key]`, `s[i]`, `len(s)` |
| O(log n) | logarithmic | [binary search](/dsa/binary-search/), balanced-tree lookup |
| O(n) | linear | one loop over a slice |
| O(n log n) | linearithmic | [`slices.Sort`](/dsa/sorting/), merge sort |
| O(n²) | quadratic | nested loops over the same input |
| O(2ⁿ) | exponential | naive recursive subsets / [Fibonacci](/dsa/dynamic-programming/) |

## Data structures — cost & when

| Structure | Access | Search | Insert | Delete | Reach for it when |
|---|---|---|---|---|---|
| [Array / slice](/dsa/arrays-and-slices/) | O(1) | O(n) | O(1)\* | O(n) | dense indexed data; the default |
| [Linked list](/dsa/linked-lists/) | O(n) | O(n) | O(1)† | O(1)† | O(1) splice at a held node (LRU) |
| [Stack / queue](/dsa/stacks-and-queues/) | — | — | O(1) | O(1) | LIFO / FIFO discipline |
| [Hash table](/dsa/hash-tables/) | — | O(1) | O(1) | O(1) | fast lookup by key, unordered |
| [BST (balanced)](/dsa/binary-search-trees/) | — | O(log n) | O(log n) | O(log n) | ordered iteration, range queries |
| [Heap](/dsa/heaps/) | O(1) peek | — | O(log n) | O(log n) | always pop the min/max next |
| [Trie](/dsa/tries/) | — | O(L) | O(L) | O(L) | prefix search, autocomplete |
| [Union-find](/dsa/union-find/) | — | O(α(n)) | O(α(n)) | — | dynamic connectivity, merge groups |

\* amortized for `append`; † given a pointer to the node; α = inverse Ackermann (≈ constant).

## Algorithm patterns

| Pattern | Use it when | Cost |
|---|---|---|
| [Binary search](/dsa/binary-search/) | input is sorted, or a monotonic predicate | O(log n) |
| [Sorting](/dsa/sorting/) | order matters; unlocks search & two-pointer | O(n log n) |
| [Two pointers](/dsa/two-pointers/) | sorted array, pair/triplet, in-place | O(n) |
| [Sliding window](/dsa/two-pointers/) | contiguous subarray / substring | O(n) |
| [BFS](/dsa/graphs/) | shortest path in hops (unweighted) | O(V + E) |
| [DFS](/dsa/graphs/) | reachability, cycles, topological order | O(V + E) |
| [Backtracking](/dsa/recursion-and-backtracking/) | enumerate candidates, undo on dead ends | exponential, pruned |
| [Dynamic programming](/dsa/dynamic-programming/) | overlapping subproblems + optimal substructure | states × transition |
| [Greedy](/dsa/greedy/) | a local choice is provably global | often O(n log n) |

## The Go standard library already has it

| Need | Reach for |
|---|---|
| Sort a slice | `slices.Sort`, `slices.SortFunc`, `sort.Slice` |
| Binary search | `slices.BinarySearch`, `sort.Search` |
| Map / set | built-in `map`; a set is `map[T]struct{}` |
| Priority queue | `container/heap` |
| Doubly linked list | `container/list` |
| Min / max of two | `min`, `max` builtins (Go 1.21) |
| Clone a slice / map | `slices.Clone`, `maps.Clone` |

## Decision shortcuts — "I need to…"

<Callout type="tip" title="✅ Pick a structure or pattern by what you're doing">
- **…look up by key, fast** → [hash table](/dsa/hash-tables/)
- **…keep data sorted / range queries** → [balanced BST](/dsa/binary-search-trees/) (or sorted slice + [binary search](/dsa/binary-search/))
- **…always take the smallest/largest next** → [heap](/dsa/heaps/)
- **…prefix match / autocomplete** → [trie](/dsa/tries/)
- **…LIFO or FIFO ordering** → [stack / queue](/dsa/stacks-and-queues/)
- **…shortest path in an unweighted graph** → [BFS](/dsa/graphs/)
- **…visit everything / detect a cycle** → [DFS](/dsa/graphs/)
- **…all combinations or permutations** → [backtracking](/dsa/recursion-and-backtracking/)
- **…optimize over overlapping subproblems** → [dynamic programming](/dsa/dynamic-programming/)
- **…a provably-optimal local choice** → [greedy](/dsa/greedy/)
- **…track connectivity / merge groups as edges arrive** → [union-find](/dsa/union-find/)
</Callout>

<Callout type="go" title="🐹 Interview reflex: brute force first, then trade space for time">
State the costs and edge cases out loud, write a correct brute force, then name the bottleneck and the pattern that removes it — a hash set turns an O(n²) duplicate scan into O(n); sorting unlocks two pointers; memoization collapses exponential recursion to polynomial. And in real Go, prefer the standard library over a hand-rolled structure unless asked to build it.
</Callout>
