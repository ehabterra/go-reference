<Callout type="analogy" title="🧅 Analogy">
Middleware is an onion around your handler. A request travels inward through each layer — logging, authentication, rate-limiting — reaches the core handler, then the response travels back out through the same layers. Each layer is the same shape (`Handler`), so you can add, remove, or reorder them freely.
</Callout>

## Routing with the standard mux

Since **Go 1.22**, `http.ServeMux` routes by method and captures path variables — so you rarely need a third-party router:

```go
mux := http.NewServeMux()
mux.HandleFunc("GET /users/{id}", getUser)      // method + wildcard
mux.HandleFunc("POST /users", createUser)
mux.HandleFunc("GET /health", health)
// inside getUser: id := r.PathValue("id")
```

More specific patterns win over less specific ones, and a trailing `/` (`/static/`) matches a subtree.

## Middleware wraps a handler

A middleware takes a `Handler` and returns a `Handler` — so they **compose** by nesting. Run it all in-process:

<Mermaid code={String.raw`graph LR
  REQ["request"] --> L["logging"] --> A["auth"] --> M["mux → handler"]
  M --> A2["auth (return)"] --> L2["logging (return)"] --> RES["response"]`} />

<Playground client:visible title="middleware.go" code={String.raw`package main

import (
	"fmt"
	"net/http"
	"net/http/httptest"
)

// middleware: same shape in, same shape out
func withRequestID(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		w.Header().Set("X-Request-ID", "req-123")
		next.ServeHTTP(w, r) // call the wrapped handler
	})
}

func main() {
	mux := http.NewServeMux()
	mux.HandleFunc("GET /users/{id}", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintf(w, "user %s", r.PathValue("id")) // path variable
	})

	handler := withRequestID(mux) // wrap the whole mux in one line

	// drive it in-process — no network needed
	rec := httptest.NewRecorder()
	handler.ServeHTTP(rec, httptest.NewRequest("GET", "/users/42", nil))

	fmt.Println("status:    ", rec.Code)                       // 200
	fmt.Println("request-id:", rec.Header().Get("X-Request-ID")) // req-123
	fmt.Println("body:      ", rec.Body.String())              // user 42
}`} />

Chain several with a small helper: `handler := logging(auth(mux))` reads outermost-first. Many codebases write a `func chain(h Handler, mw ...Middleware) Handler` to apply a slice in order.

<Callout type="tip" title="🐹 Middleware is just composition — keep it boring">
The whole pattern is `func(http.Handler) http.Handler`. Resist a framework until the standard mux genuinely can't express your routes. Common middlewares: request logging, panic [recovery](/fundamentals/panic-recover/) (so one bad handler can't crash the server), auth, CORS, gzip, and [rate limiting](/patterns/semaphore/). Put cross-cutting concerns here and keep handlers focused on one job.
</Callout>
