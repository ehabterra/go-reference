<Callout type="analogy" title="🔌 Analogy">
`fmt` is the label-maker: it turns values into readable text. `io.Reader` and `io.Writer` are the **universal plugs** — every source of bytes (a file, a socket, a string) is a Reader, every sink (a file, the terminal, a buffer) is a Writer, and because the plugs are standard, anything connects to anything.
</Callout>

## Printing with fmt

`Println` is for quick output; `Printf` gives you **verbs** for control; `Sprintf` returns a string instead of printing:

<Playground client:visible title="fmt.go" code={String.raw`package main

import "fmt"

type Point struct{ X, Y int }

func main() {
	p := Point{1, 2}
	fmt.Printf("%v\n", p)   // {1 2}           — default
	fmt.Printf("%+v\n", p)  // {X:1 Y:2}        — with field names
	fmt.Printf("%#v\n", p)  // main.Point{X:1, Y:2} — Go syntax
	fmt.Printf("%T\n", p)   // main.Point       — the type

	fmt.Printf("%d  %s  %q  %.2f\n", 42, "hi", "hi", 3.14159)
	//          42  hi  "hi"  3.14

	s := fmt.Sprintf("(%d, %d)", p.X, p.Y) // build, don't print
	fmt.Println("built:", s)
}`} />

The verbs you'll use most: `%v` (anything), `%d` (integer), `%s` (string), `%q` (quoted), `%f`/`%.2f` (float), `%t` (bool), `%T` (type), `%w` (wrap an [error](/fundamentals/errors/)).

## The io interfaces

Two one-method interfaces underpin all of Go's I/O:

<Mermaid code={String.raw`graph LR
  SRC["any source<br/>file · socket · string"] -->|"Read(p []byte)"| R["io.Reader"]
  R -->|"io.Copy"| W["io.Writer"]
  W -->|"Write(p []byte)"| DST["any sink<br/>file · terminal · buffer"]`} />

Because files, network connections, `strings.Reader` and `bytes.Buffer` all satisfy them, `io.Copy` streams from **any** Reader to **any** Writer — and `fmt` uses a type's `String()` method (the **Stringer** interface) automatically:

<Playground client:visible title="io.go" code={String.raw`package main

import (
	"fmt"
	"io"
	"strings"
)

// Implementing Stringer controls how fmt prints this type.
type Celsius float64

func (c Celsius) String() string {
	return fmt.Sprintf("%.1f°C", float64(c))
}

func main() {
	// any io.Reader copied into any io.Writer
	src := strings.NewReader("hello, io")
	var dst strings.Builder // strings.Builder is an io.Writer
	n, _ := io.Copy(&dst, src)
	fmt.Printf("copied %d bytes: %q\n", n, dst.String())

	fmt.Println("temp:", Celsius(36.6)) // calls String() -> 36.6°C
}`} />

## Buffer I/O for speed

Wrapping a Reader/Writer in `bufio` batches tiny reads and writes. `bufio.Scanner` is the idiomatic way to read input line by line:

```go
scanner := bufio.NewScanner(os.Stdin)
for scanner.Scan() {
	line := scanner.Text()
	// ... process line
}
```

<Callout type="tip" title="🐹 Write to an io.Writer, not to the screen">
Functions that produce text should take an `io.Writer` (or use `fmt.Fprintf(w, …)`) instead of calling `fmt.Println` directly. Then the same code writes to the terminal, a file, an HTTP response, or a `bytes.Buffer` in a test — no changes. This is the whole reason `os.Stdout` is just an `*os.File` that happens to be a Writer.
</Callout>
