<Callout type="analogy" title="✂️ Analogy">
If a `string` is a sealed, read-only ribbon of text, the `strings` package is the toolbox that *measures and cuts copies* of it — find, split, replace, trim — without ever altering the original. When you need to **assemble** text instead, `strings.Builder` is the workbench; when you're holding mutable bytes, `bytes` is the same toolbox for a different material.
</Callout>

## The strings package

A flat set of functions over immutable strings — search, split, transform:

<Playground client:visible title="strings.go" code={String.raw`package main

import (
	"fmt"
	"strings"
)

func main() {
	s := "  Go is fun  "
	fmt.Printf("%q\n", strings.TrimSpace(s))          // "Go is fun"
	fmt.Println(strings.ToUpper("go"))                // GO
	fmt.Println(strings.Contains(s, "fun"))           // true
	fmt.Println(strings.Split("a,b,c", ","))          // [a b c]
	fmt.Println(strings.Join([]string{"x", "y"}, "-")) // x-y
	fmt.Println(strings.ReplaceAll("aaa", "a", "b"))  // bbb
	fmt.Println(strings.HasPrefix("gopher", "go"))    // true
	fmt.Println(strings.Count("banana", "a"))         // 3
}`} />

## Build, don't concatenate

A `string` is **immutable**, so `s += more` allocates a whole new string every time — quadratic in a loop. `strings.Builder` writes into one growing buffer:

<Playground client:visible title="builder.go" code={String.raw`package main

import (
	"fmt"
	"strings"
)

func main() {
	var b strings.Builder
	for i := 0; i < 5; i++ {
		fmt.Fprintf(&b, "item-%d ", i) // Builder is an io.Writer
	}
	out := b.String()
	fmt.Printf("%q (len %d)\n", strings.TrimSpace(out), b.Len())
}`} />

## bytes and strconv

<Mermaid code={String.raw`graph LR
  STR["string (immutable)"] <-->|"[]byte(s) / string(b)"| BYT["[]byte (mutable)"]
  STR -->|"strconv.Atoi / Itoa"| NUM["int, float, bool"]
  STR --- P1["strings.* funcs"]
  BYT --- P2["bytes.* funcs (mirror)"]`} />

The `bytes` package mirrors `strings` for `[]byte` (handy for I/O buffers, where `bytes.Buffer` is the mutable cousin of `strings.Builder`). `strconv` converts between strings and numbers — and always returns an error to check:

```go
n, err := strconv.Atoi("42")   // string -> int
s := strconv.Itoa(42)          // int -> string
f, err := strconv.ParseFloat("3.14", 64)
b, err := strconv.ParseBool("true")
```

<Callout type="gotcha" title="⚠️ Converting string ↔ []byte copies">
`[]byte(s)` and `string(b)` each **allocate and copy** — strings are immutable, so Go can't share the memory safely. That's usually fine, but in a hot loop, work in one representation (stay in `[]byte` for I/O, `string` for keys) rather than converting back and forth. For character-by-character work, remember `string` indexing gives **bytes**, not characters — see [strings & runes](/fundamentals/strings-and-runes/).
</Callout>
