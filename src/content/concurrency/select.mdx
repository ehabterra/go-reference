<Callout type="analogy" title="🎛️ Analogy">
`select` is a switchboard operator watching several lines. Whichever line rings first, they answer it. If two ring at once, they pick one at random. With a "default" instruction, they don't wait at all — if no line is ringing this instant, they move on.
</Callout>

## What select does

`select` blocks until **one** of its channel operations can proceed, then runs that case. It's how you compose multiple channels — receives, sends, timeouts, cancellation — into one coordination point.

```go
select {
case v := <-c1:
	use(v)
case c2 <- x:        // a send can be a case too
	sent()
case <-done:
	return            // cancellation
}
```

## Three behaviors to know

- **Random choice:** if several cases are ready, one is picked **uniformly at random** — this prevents starvation.
- **`default` → non-blocking:** with a `default`, select returns immediately if nothing is ready.
- **Empty `select{}` blocks forever** — occasionally used to park `main` while goroutines run (but usually a `sync.WaitGroup` or signal is clearer).

## Timeout and non-blocking, in one run

<Playground client:visible title="select.go" code={String.raw`package main

import (
	"fmt"
	"time"
)

func main() {
	ch := make(chan string)
	go func() {
		time.Sleep(50 * time.Millisecond)
		ch <- "result"
	}()

	// Timeout pattern: whichever fires first wins.
	select {
	case v := <-ch:
		fmt.Println("got:", v)
	case <-time.After(100 * time.Millisecond):
		fmt.Println("timed out")
	}

	// Non-blocking check: don't wait at all.
	select {
	case v := <-ch:
		fmt.Println("got:", v)
	default:
		fmt.Println("nothing ready right now")
	}
}`} />

<Callout type="go" title="🐹 Two sharp edges">
**`time.After` in a hot loop leaks timers** — each call allocates a timer that lives until it fires. In a tight `for { select … }`, create one `time.NewTimer` and `Reset` it, or drive cancellation with a `context`. And remember: **a `nil` channel blocks forever**, so setting a channel variable to `nil` *disables* its `select` case — a handy trick for turning a branch off once it's done.
</Callout>
